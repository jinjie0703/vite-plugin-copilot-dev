# Test-Driven Agent Skill

为了彻底阻绝 Vibe Coding 过程中的核心逻辑退化（Regression），必须采取测试驱动的兜底策略。

## 触发条件

在进行以下非纯视觉的底层修改时：

1. 核心数据与状态层（如 `mcp/context.ts` 的重构，缓存队列更新机制）。
2. 网络通信与中间件层（MCP 传输层连接逻辑、WebSocket 拦截与重试机制）。
3. 解析与数据处理层（SourceMap 错误堆栈的解析、AST 提取逻辑）。

## 执行步骤

1. **寻找防护网**：
   - 检查即将修改的代码旁边是否有 `.test.ts` 或 `.spec.ts` 文件。
   - 如果没有，不要嫌麻烦，立刻帮人类写一个极简的边界测试（Vitest / Jest）。
2. **测试驱动修改**：
   - 先写出预期会失败的测试（针对新功能或新结构）。
   - 在终端后台运行 `npm run test` 等效命令，确认其处于红灯（Fail）状态。
   - 动手修改业务代码。
3. **绿色通行证**：
   - 再次运行测试，直到测试输出全绿（Pass）。
   - 如果全绿，带着测试日志和人类汇报：“我已经帮你用自动化测试兜底过了，这个逻辑绝对稳如泰山。”
   - 绝不允许把没有跑通测试的核心业务代码扔给人类。
