# Context & README Sync Skill

为了防止 Vibe Coding 过程中文档逐渐与代码脱节、产生“幻觉”，必须建立强大的文档同步更新机制。

## 触发条件

1. 完成了一个大版本更新、新增了重要特性。
2. 架构模式发生了根本改变（例如：重构了 Vite 生命周期钩子的逻辑、修改了 MCP 传输层通信协议）。
3. 用户明确要求更新文档。

## 执行步骤

1. **回顾上下文**：梳理刚才所完成的核心改动（新增了哪些库？改了什么核心机制？）。
2. **README 强制同步**：
   - 打开 `README.md`。
   - 寻找 `## 🌟 核心亮点` 或其他相关章节。
   - 将新特性的总结（1-2句简练专业的话）写入其中。
3. **架构蒸馏更新 (Architecture Distillation)**：
   - 检查是否存在 `.agents/architecture_map.md`。如果没有，则自动创建一个。
   - 用精简的 Markdown 记录当前的核心数据流、组件划分或接口约定（无需事无巨细，只写“骨架”）。
   - 让未来的对话只要看这个文件，就能瞬间回忆起整个项目的运转方式。
4. **汇报**：告诉用户文档和 README 已全部同步完毕。
