# Self-Healing Build Skill

在 Vibe Coding 模式中，人类只负责给出意图，解决语法和类型报错是 AI 自己的责任。

## 触发条件

1. 在运行 `npm run build` (tsup)、`npm run dev` 或 `tsc` 等构建命令时发现 Error 输出。
2. 任何涉及到强类型系统（如 TypeScript）改动后，代码编辑器报出红色的错误诊断。

## 执行步骤

1. **拦截阻断**：绝不可以将报错的完整堆栈抛给人类，然后说“请看这个报错”。你必须自己接管修复任务。
2. **错误解析与搜寻**：
   - 阅读构建输出中的具体错误信息。
   - 使用工具（如 `view_file` 或 `grep_search`）定位到报错的具体行数和上下文。
3. **尝试修复与类型对齐**：
   - 注意：如果是 TypeScript 中提示 `any` 相关的错误，请严防死守，绝不能用 `@ts-ignore` 敷衍了事，必须使用泛型或 `unknown` 进行安全的类型推导。
   - 根据工业界最佳实践修改错误代码。
4. **验证循环 (The Loop)**：
   - 修复后，立刻**重新运行刚才失败的构建命令**。
   - 如果仍然报错，跳回步骤 2 继续修。此过程允许后台重试，直到输出为 0 Errors。
5. **最终汇报**：成功后，告诉用户你悄悄修复了哪些棘手的类型问题，目前构建状态已完美绿灯。
