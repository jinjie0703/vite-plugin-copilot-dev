# Auto Snapshot Skill

在 Vibe Coding 模式下，代码变更速度极快。为了防止“翻车”导致无法挽回的局面，本技能强制在危险操作前留档。

## 触发条件

当你要执行以下操作前：

1. 跨越 3 个及以上文件的大型重构。
2. 删除旧模块、重命名大量核心组件。
3. 升级或更改核心依赖（如 `package.json` 中的构建工具或 MCP SDK）。

## 执行步骤

1. **状态检查**：运行 `git status` 检查当前工作区。
2. **强制暂存**：如果工作区有未提交的代码，先运行 `git add .`。
3. **快照提交**：运行 `git commit -m "chore: snapshot before risky operations [Auto]"`。
4. **提示用户**：明确告诉用户：“我已经为您做好了本地防回滚快照，现在开始执行重构。如果后续结果不满意，您可以随时回滚到这个状态。”
5. 提交成功后，再去真正执行你计划的改代码操作。
